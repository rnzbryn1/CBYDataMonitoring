Implementation Summary

How to Test

  1. Open the main application (any of the HTML files like pcd.html)
  2. Select a template with multiple columns
  3. Check table headers: You should see variable names (A, B, C...) with column names as smaller text below
  4. Right-click on any cell and select "Compute"
  5. In the compute modal:
    • Column buttons should show variable names prominently
    • Formula placeholder should show "=A * B" format
    • Clicking column buttons inserts variable names
  6. Test formulas:
    • Enter "=A * B" and apply (where A and B are your column variables)
    • The formula should work correctly
    • Re-open the compute modal - it should show the variable formula
  7. Test functions:
    • Try "=SUM(A, B)" or "=AVERAGE(A, B)"
    • Functions should work with variable names

Expected Features

  • Variable Generation: A, B, C, ..., Z, AA, AB, etc. (Excel-style)
  • Display Format: Variable name shown prominently, column name as subtitle
  • Formula Input: Users type variables instead of full column names
  • Backward Compatibility: Existing formulas with column names still work
  • Automatic Conversion: System converts between variables and column names as needed

Technical Implementation Details

New State Variables:

columnVariables: {}  // { "columnName": "A", "columnName2": "B", ... }
variableColumns: {}  // { "A": "columnName", "B": "columnName2", ... }

Key Functions Added:

generateColumnVariables()     // Creates A,B,C mappings
indexToColumnVariable(index)   // Converts index to Excel-style column name
getColumnVariable(columnName)  // Gets variable for column name
getColumnNameFromVariable(variable)  // Gets column name from variable
convertFormulaToVariables(formula)   // Converts formula to use variables
convertFormulaToColumnNames(formula)  // Converts formula back to column names

Updated Components:

  • Compute Modal: Shows variables, inserts variables on button click
  • Table Headers: Display variables with column names as subtitles
  • Formula Evaluation: Supports both variables and column names
  • Template Switching: Clears and regenerates variable mappings
