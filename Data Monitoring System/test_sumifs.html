SUMIFS Function Test & Documentation

🎯 SUMIFS Function Overview

The SUMIFS function has been implemented for your monitoring category. It works like Excel's SUMIFS but pulls data directly from your database and caches results for performance.

Important: The function is case-insensitive and automatically caches results to improve performance.

📝 Syntax & Examples

Basic Syntax:
=SUMIFS('sum_column', 'condition_column1', 'condition_value1', 'condition_column2', 'condition_value2', ...)
Example 1 - Single Condition:
=SUMIFS('Accepted QTY', 'Item Description', 'Banana Saba')
Example 2 - Multiple Conditions:
=SUMIFS('Accepted QTY', 'Item Description', 'Banana Saba', 'Month #', '10')
Example 3 - Three Conditions:
=SUMIFS('Accepted QTY', 'Item Description', 'Banana Saba', 'Month #', '10', 'Year', '2024')

🔧 How It Works

  1. Database Query: The function searches through all encoding templates in your department
  2. Condition Matching: It finds entries that match ALL specified conditions (AND logic)
  3. Sum Calculation: It sums the values in the specified sum column for matching entries
  4. Caching: Results are cached to improve performance on subsequent evaluations
  5. Auto-Refresh: The UI automatically updates when cached results are available

⚡ Performance Features

  • Smart Caching: Results are cached based on department, sum column, and conditions
  • Async Processing: Database queries run asynchronously to prevent UI freezing
  • Case-Insensitive: All text comparisons are case-insensitive
  • Error Handling: Returns 0 instead of errors if no data is found

🚀 Usage in Monitoring Category

To use SUMIFS in your monitoring category:

  1. Navigate to your monitoring template
  2. Select a cell or column where you want the formula
  3. Enter the formula using the syntax above
  4. The system will automatically pull data from your encoding sheets
  5. Results will update automatically when data changes
